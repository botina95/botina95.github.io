🏗️ Arquitectura

Tres componentes. Un ecosistema.

CAPTOR se distribuye en tres aplicaciones que trabajan en conjunto: el diseñador de flujos, el servidor central y el ejecutor que usa el operario en su puesto de trabajo.

🎨
Administrador

Captor Designer

Aplicación de escritorio para el equipo técnico. Aquí se diseñan los perfiles, se configuran los proveedores de datos, se gestionan usuarios, roles, grupos y se monitorean los logs de ejecución y auditoría de seguridad.

🖥️
Infraestructura

Captor Server

Servicio Windows con API REST (ASP.NET Core). Centraliza perfiles, usuarios y sesiones. Soporta autenticación JWT, Active Directory, HTTPS opcional, rate limiting y control de sesiones concurrentes según licencia.

⚡
Operario

Captor Executor

Cliente ligero en la bandeja del sistema. El operario activa los perfiles con un hotkey global sin salir de su aplicación. Captura datos en pantalla, ejecuta la consulta y muestra el resultado al instante.

🔌 Proveedores de Datos

7 fuentes de datos. Sin límites.

Conecta CAPTOR directamente a tu base de datos, tus APIs internas o externas, archivos Excel, texto plano o lógica personalizada en C#. Cada proveedor se configura visualmente sin tocar código.

🗄️

SQL Server

Ejecuta queries parametrizados directamente sobre tu base de datos SQL Server corporativa. Compatible con autenticación Windows e integrada.

🐬

MySQL / MariaDB

Conéctate a bases de datos MySQL o MariaDB con la misma simplicidad. Ideal para sistemas de gestión internos o aplicaciones web empresariales.

📦

SQLite

Para catálogos locales o escenarios offline. La base de datos SQLite viaja con el perfil y no requiere servidor adicional.

🌐

API REST

Consume cualquier API externa o interna. Soporta Bearer token, Basic Auth y token dinámico obtenido de un endpoint de login. Headers y body JSON o Form configurables visualmente.

📊

Excel

Lee archivos .xlsx directamente como fuente de datos. Útil para catálogos de productos, tarifas o cualquier tabla gestionada por el equipo sin BD.

📄

Archivo de texto

Consulta archivos CSV o TXT delimitados. Compatible con exportaciones de sistemas legacy que no tienen API ni BD accesible.

⚙️

Script C#

Lógica completamente libre con Roslyn. Compilación dinámica, testing con F5 desde el diseñador, syntax highlighting y acceso completo a .NET para cualquier integración personalizada.

📤 Tipos de Respuesta

8 formas de entregar el resultado.

Una vez obtenida la información, CAPTOR puede presentarla, escribirla, abrirla o encadenarla de 8 maneras diferentes según el flujo de trabajo del operario.

💬

Mensaje

Muestra un texto simple al operario: confirmación, advertencia o dato consultado. Ideal para validaciones rápidas en pantalla.

📋

Tabla

Presenta el resultado como grilla de datos. El operario puede revisar múltiples registros y seleccionar el que necesita.

⌨️

Escritura en campos

CAPTOR escribe automáticamente los valores en los campos de la aplicación activa usando UIA o SendKeys. El operario no tiene que copiar ni pegar nada.

🔧

Script C#

Lógica personalizada para transformar, combinar o enrutar el resultado antes de presentarlo. Sin restricciones en la manipulación de datos.

🔗

Combinado

Encadena múltiples acciones en secuencia: consultar, escribir campos y mostrar un mensaje, todo en un solo perfil.

🌍

Abrir URL

Abre automáticamente una URL construida con los datos capturados. Perfecto para abrir expedientes en sistemas web sin buscar manualmente.

📁

Abrir carpeta

Navega directamente a la carpeta del sistema de archivos correspondiente al registro procesado. Cero búsqueda manual en el explorador.

📂

Abrir archivo

Localiza y abre el archivo específico asociado al resultado de la consulta. Ideal para acceder a documentos, imágenes o PDFs de expedientes.

🔒 Seguridad

Diseñado para entornos corporativos.

CAPTOR implementa un modelo de seguridad por capas pensado para empresas con requisitos de control de acceso, trazabilidad y cumplimiento.

🎫

JWT + Active Directory

Autenticación con tokens JWT de corta duración. Soporte para Active Directory con validación de credenciales corporativas o SSO automático por dominio.

👥

Roles y grupos de perfiles

Define qué perfiles puede ejecutar cada usuario o grupo. Control granular por rol: diseñador, operario, administrador.

📊

Log de ejecuciones

Cada ejecución queda registrada: usuario, perfil, fecha/hora, datos capturados y resultado. Trazabilidad completa de operaciones.

🛡️

Auditoría de seguridad

Registro de todos los eventos de seguridad: logins exitosos, intentos fallidos, cierres de sesión y cambios de contraseña con IP y detalle.

🔄

Sesiones concurrentes controladas

La licencia define el máximo de usuarios simultáneos. Las sesiones se monitoran en tiempo real con heartbeat automático.

🌐

HTTPS configurable

Soporte para certificado .pfx propio. Configurable desde el panel de setup del servidor: puede usarse en HTTP para intranets o HTTPS para redes públicas.

☁️

Validación de licencia en nube

Cada licencia se verifica periódicamente contra la nube. Detección de manipulación del reloj del sistema y período de gracia controlado para entornos con conectividad limitada.

📝

Historial de cambios

Registra cada modificación de configuración: quién cambió qué, valor anterior y valor nuevo. Ideal para cumplimiento normativo.
